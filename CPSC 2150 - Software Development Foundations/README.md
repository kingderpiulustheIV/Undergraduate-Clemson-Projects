# CPSC 2150 - Software Development Foundations

## Course Description
Introduction to software engineering principles and practices. Focus on software design patterns, testing methodologies, version control, and collaborative software development. Students work in teams to design, implement, and test larger-scale applications using industry-standard practices.

## Learning Objectives
- Apply software engineering principles and design patterns
- Practice test-driven development (TDD) using JUnit
- Master object-oriented design and UML modeling
- Develop proficiency with version control systems (Git)
- Collaborate effectively in team-based software projects
- Implement GUI applications using JavaFX
- Write comprehensive unit tests and conduct code reviews
- Understand software development lifecycle and agile methodologies

## Projects and Assignments

### Project 1: Designing Extended ConnectX
Design and implementation of an extended Connect Four game with customizable board sizes and win conditions. Focus on object-oriented design, interfaces, and multiple implementations.

### Project 2
Continuation of software development practices with emphasis on testing and design patterns.

### Project 3
Advanced software engineering project incorporating multiple design patterns and complex system architecture.

### Project 5: Testing Our Projects
Comprehensive testing project focusing on unit testing, integration testing, and test coverage using JUnit 5.

### Project 6: Adding a Java Swing GUI
GUI development project implementing graphical interfaces for previous console-based applications using Java Swing.


## Technology Stack

### Programming Language
- **Java**: Primary language for all coursework

### Frameworks and Libraries
- **JavaFX**: GUI development framework
- **JUnit 5**: Unit testing framework
- **Java Swing**: Alternative GUI framework (Project 6)

### Development Tools
- Java Development Kit (JDK 8 or later)
- IntelliJ IDEA
- Git version control
- Maven or Gradle (build automation)
- JUnit for automated testing

### Design Practices
- UML Class Diagrams
- Design Patterns (Factory, Singleton, Strategy, Observer, etc.)
- Object-Oriented Design Principles (SOLID)
- Test-Driven Development (TDD)
- Code Review and Pair Programming

### Environment
- Cross-platform Java development
- Command-line and IDE-based development
- Windows 10 x86-64 

## Setup and Requirements
- Java JDK 8 or later
- IDE (IntelliJ IDEA or Eclipse recommended)
- Git for version control
- JavaFX SDK (if not included in JDK)
- JUnit 5 testing framework
- Basic understanding of OOP concepts

## Work Type
This class is **group-based collaborative work** using Java, JavaFX, and JUnit 5.
